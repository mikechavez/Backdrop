# Adapt Article for X (Native Article or Thread)

## Objective
Create an X-native version of the piece. Two options — decide which is better for reach, then execute.

## Context
X offers two native long-form options: Articles (long-form posts on X) and threads. Both get better algorithmic treatment than an external link to Substack. The main tweet will link to the interactive site — this piece is the deeper companion.

## Tool Routing
- Tool: Claude Web
- Model: N/A

## Execution Steps

### Decision: Article vs Thread

**X Article (pros):**
- Clean reading experience
- Feels substantial and professional
- Can include images inline
- Single link to share

**X Thread (pros):**
- Each tweet is individually engageable (likes, RTs, replies)
- Algorithm shows threads to more people
- Can go viral tweet-by-tweet
- More "native" to how X works

**Recommendation:** Do both. Write a 5–7 tweet thread as the primary distribution vehicle (TASK-008 covers this). Write a shorter X Article as supplementary content for people who want more depth but won't click to Substack.

### If X Article:
- 800–1,000 words (shorter than LinkedIn version)
- Lead with the story, not the concept
- Punchier, more opinionated tone than LinkedIn
- End with link to interactive site
- Include 1–2 screenshots of the simulator

### If Thread Only:
- Handled in TASK-008 (launch copy)
- Skip this ticket
