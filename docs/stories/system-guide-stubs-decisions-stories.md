# Decision Record Template

> Only write if there's real tension, tradeoff, or future-you might disagree.
> Skip: "we used X library", "we set this to 0.5", "we named it foo"
> Write: bets you might regret, real tradeoffs, multi-system impact

---

## Metadata

**Date:** (YYYY-MM-DD)

**Decision:** (one sentence, clear what was decided)

**Status:** DECIDED / OPEN / SUPERSEDED

**Revisit:** (Never / 2027-Q2 / After we see X metric / etc)

**Related story:** (if this is in a product story, link it here. Otherwise "none yet")

---

## Context

*Why did this decision come up? What problem forced it?*

(2-3 paragraphs max. Be concise.)

---

## The Decision

*What did you actually decide?*

(Be specific. If there were options, list them briefly.)

---

## Why This Over Alternatives?

*What was the tradeoff you accepted?*

(This is the key part. Why this choice, not the other one?)

---

## How We'll Know If We Got It Right

*What metric or signal proves this was a good call?*

(If you can't articulate this, the decision wasn't tight enough.)

---

## Related Decisions

*What other decisions does this one link to or depend on?*

(Optional, but helpful for navigation.)

---

## Caveats

*What could go wrong? What assumptions might break?*

(Keep it real.)

---

## Constraint

**Do not write this if:**
- It's obvious in hindsight
- No real tradeoff exists
- It's a technical implementation detail
- You can't imagine disagreeing with it later

**Do write this if:**
- There's tension between options
- You're making a bet about the future
- It affects multiple systems
- You might regret it in 6 months
