02 — Failure Modes

The invisible ways agents fail

These aren't hypothetical. Every failure below happened during a real six-month build. They represent the most common ways AI agents break production systems — failures that don't show up in demos, only in sustained development.

Click a card to simulate the failure in real time. Then toggle "With Context Engineering" to see how the same situation plays out when you have the right structure in place.

Failure 01

Token Burn

Agent interprets "verify" as "investigate" and burns ~50,000 tokens reading files it never needs.
Failure 02

Hallucinated Metrics

Agent logs implementation time as 2 hours. Actual time: 10 minutes. Fabricated with full confidence.

Duplicate Narratives

Provenance required. The context engineering spec mandates all metrics include a source field. If the agent can't measure it, it must say so. Fabrication caught at the template level.

Narrative fingerprinting + dedup thresholds. LLM judges if two articles describe the same event. Deterministic code scores match confidence and merges above threshold. 89% accuracy. Zero duplicates shown to users.

The core lesson
AI doesn't fail because it's incapable. It fails because it has no coherent model of your system, your constraints, or your intent. Supplying that coherence is your job.
03 — The Fix

Context Engineering

Context engineering is the discipline of structuring what your AI agent knows at the start of every session — your architecture, constraints, past decisions, and the reasoning behind them.

Without it, every new session starts from absolute zero. The agent has no memory of your project. Watch the difference below.
